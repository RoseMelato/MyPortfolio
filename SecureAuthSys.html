C# ASP.NET Core JWT Authentication SQL Server REST API

Project Overview

Problem Statement

Modern applications require secure authentication systems to protect user data and restrict unauthorized access. Many beginner-level authentication systems suffer from common security problems:

  • Storing passwords in plain text
  • Lack of authentication tokens
  • Insecure login systems
  • No password recovery mechanism
  • Poor user identity verification
Security Risk: Without proper security mechanisms, applications are vulnerable to unauthorized access, data breaches, identity theft, and session hijacking.

My Solution

A secure authentication REST API built with ASP.NET Core that implements modern security best practices:

  • JWT-based authentication for secure login
  • Password hashing (never stored in plain text)
  • User registration with validation
  • Password reset functionality with token verification
  • Protected API endpoints with role-based access
  • SQL Server database with Entity Framework Core

Key Code Implementations

JWT Token Generator

C# 
public class JwtHelper
{
    private readonly IConfiguration _configuration;
    
    public JwtHelper(IConfiguration configuration)
    {
        _configuration = configuration;
    }
    
    public string GenerateToken(User user)
    {
        var securityKey = new SymmetricSecurityKey(
            Encoding.UTF8.GetBytes(_configuration["Jwt:Key"]));
        var credentials = new SigningCredentials(
            securityKey, SecurityAlgorithms.HmacSha256);
        
        var claims = new[]
        {
            new Claim(JwtRegisteredClaimNames.Sub, user.Id.ToString()),
            new Claim(JwtRegisteredClaimNames.Email, user.Email),
            new Claim(ClaimTypes.Name, user.Username),
            new Claim(ClaimTypes.Role, user.Role),
            new Claim(JwtRegisteredClaimNames.Jti, Guid.NewGuid().ToString())
        };
        
        var token = new JwtSecurityToken(
            issuer: _configuration["Jwt:Issuer"],
            audience: _configuration["Jwt:Audience"],
            claims: claims,
            expires: DateTime.UtcNow.AddHours(2),
            signingCredentials: credentials
        );
        
        return new JwtSecurityTokenHandler().WriteToken(token);
    }
}

Password Hasher

C# 
public class PasswordHasher
{
    public string HashPassword(string password)
    {
        return BCrypt.Net.BCrypt.HashPassword(password);
    }
    
    public bool VerifyPassword(string password, string passwordHash)
    {
        return BCrypt.Net.BCrypt.Verify(password, passwordHash);
    }
}

// Usage during registration:
var hashedPassword = _passwordHasher.HashPassword(password);

// Usage during login:
var isValid = _passwordHasher.VerifyPassword(password, user.PasswordHash);

Login Endpoint

C# 
[HttpPost("login")]
public async Task<IActionResult> Login(LoginDto model)
{
    // Find user by email
    var user = await _context.Users
        .FirstOrDefaultAsync(u => u.Email == model.Email);
    
    if (user == null)
        return Unauthorized("Invalid email or password");
    
    // Verify password hash
    bool isPasswordValid = _passwordHasher.VerifyPassword(
        model.Password, user.PasswordHash);
    
    if (!isPasswordValid)
        return Unauthorized("Invalid email or password");
    
    // Generate JWT token
    var token = _jwtHelper.GenerateToken(user);
    
    return Ok(new
    {
        token = token,
        user = user.Username,
        role = user.Role,
        expiresIn = 7200 // 2 hours in seconds
    });
}

Challenges & Solutions

JWT Key Size Error

An error occurred due to a JWT secret key that was too short. The HS256 algorithm requires a key of at least 256 bits (32 characters).

Solution:

Increased the secret key length in configuration to meet the minimum requirement. Used a secure, randomly generated key of sufficient length.

Password Reset Token Issue

An error occurred during password reset due to database constraints when saving null values for reset token fields.

Solution:

Ensured database fields allowed null values and properly cleared reset tokens after successful password updates by setting them to null.

Token Expiry Management

Reset tokens needed to expire after a certain time for security, but also needed to be invalidated after use.

Solution:

Implemented token expiry validation checking both that the token exists and that the current time is before the expiry time. Tokens are cleared after successful use.
