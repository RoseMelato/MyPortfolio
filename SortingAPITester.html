Sorting API Tester

A Web-Based Tool for Testing Sorting Algorithm APIs

My Role: Full-Stack Developer
Duration: 1 week
Project Type: Individual Project
Status: Completed

Frontend Development API Integration User Interface Design JSON Processing

Project Overview

The Challenge

Testing sorting APIs can be tedious when working with command-line tools or complex API clients. Developers need a simple, visual way to test sorting endpoints without writing code or using advanced tools.

  • No simple interface for testing sorting APIs
  • Command-line tools are not user-friendly
  • Need for immediate visual feedback
  • API responses should be easy to read
  • Quick testing of different input strings

My Solution

A clean, web-based interface that allows users to test sorting APIs with a simple form, showing the response in a readable JSON format with the array displayed on a single line.

  • Simple form-based API testing
  • Clean JSON response display
  • Single-line array formatting
  • Error handling and validation
  • Responsive design for all devices

Key Code Implementation

API Request Handler

JavaScript 
// Simple API testing function
document.getElementById("testBtn").addEventListener("click", async () => {
    const email = document.getElementById("email").value.trim();
    const apiUrl = document.getElementById("apiUrl").value.trim();
    const data = document.getElementById("data").value.trim();

    if (!email || !apiUrl || !data) {
        alert("Please fill in all fields!");
        return;
    }

    try {
        document.getElementById("response").textContent = "Loading...";
        
        const res = await fetch(apiUrl, {
            method: "POST",
            headers: { "Content-Type": "application/json" },
            body: JSON.stringify({ data })
        });

        const json = await res.json();
        
        // Format the response to show array in one line
        let formattedResponse;
        if (json.word && Array.isArray(json.word)) {
            formattedResponse = `{\n  "word": [${json.word.map(item => `"${item}"`).join(', ')}]\n}`;
        } else {
            formattedResponse = JSON.stringify(json, null, 2);
        }
        
        document.getElementById("response").innerHTML = formattedResponse;
          
    } catch (err) {
        document.getElementById("response").innerHTML = "Error: " + err.message;
    }
});

How It Works

1

User Input

User enters their email, the API endpoint URL, and the string they want to sort in the input fields.

2

API Request

The application sends a POST request to the specified API with the string data in JSON format: {"data": "example"}

3

Response Processing

The API response is received and processed. The application expects a JSON response with a word property containing an array.

4

Formatted Display

The response is formatted to show the array on a single line, making it easy to read: {"word": ["a", "e", "e", "l", "m", "p", "x"]}

Challenges & Solutions

JSON Formatting Issue

The default JSON.stringify() method displays arrays with each element on a new line, making the output harder to read for simple arrays.

Solution:

Created custom formatting logic that maps through the array and joins elements with commas on a single line: json.word.map(item => `"${item}"`).join(', ')

Different API Response Formats

Different APIs might return data in various formats: array only, object with word property, string, or other structures.

Solution:

Implemented flexible response handling that checks the response type and formats accordingly, ensuring consistent output regardless of the API's format.

Mobile Responsiveness

The interface needed to be usable on mobile devices where screen space is limited.

Solution:

Used CSS flexbox and media queries to create a responsive layout that adapts to all screen sizes, with appropriately sized input fields and buttons.
